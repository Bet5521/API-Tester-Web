# 通用接口调试工具V3.2 - 操作手册

## 1. 系统概览

通用接口调试工具是一个专业的HTTP接口测试工具，提供了直观的用户界面和丰富的功能，帮助开发人员和测试人员快速测试和调试API接口。

### 1.1 系统布局

- **页头**：包含系统名称和导航菜单
- **内容区域**：根据选择的导航页签显示不同的功能模块
- **页脚**：显示版权信息

### 1.2 导航菜单

- **接口测试**：用于测试HTTP接口
- **接口管理**：用于管理接口配置
- **历史记录**：查看过往请求历史
- **设置**：配置系统参数
- **README**：查看系统简介
- **操作手册**：查看详细使用指南

## 2. 接口测试

### 2.1 基本操作

1. **选择接口**：从下拉菜单中选择已保存的接口，或直接输入接口信息
2. **设置请求方法**：从下拉菜单中选择GET、POST、PUT或DELETE
3. **输入请求URL**：输入完整的接口URL
4. **添加请求参数**：点击"添加参数"按钮添加URL参数
5. **添加请求体参数**：点击"添加参数"按钮添加请求体参数
6. **编辑请求体JSON**：在右侧文本框中直接编辑JSON格式的请求体
7. **添加请求头**：点击"添加请求头"按钮添加请求头
8. **发送请求**：点击"发送请求"按钮发送HTTP请求
9. **查看响应**：在下方查看响应状态、响应时间和响应结果

### 2.2 高级功能

#### 2.2.1 JSON类型参数编辑

1. 在请求体参数列表中，添加一个参数并将类型设置为"JSON"
2. 点击"编辑"按钮打开JSON编辑器
3. 在JSON编辑器中，选择JSON类型（对象或数组）
4. 添加或编辑子参数
5. 如果选择了数组类型，可以添加多个参数组
6. 点击"保存"按钮保存JSON参数

#### 2.2.2 请求体与参数列表联动

- 修改请求体参数列表中的参数，右侧JSON文本框会自动更新
- 修改右侧JSON文本框中的内容，左侧参数列表会自动更新

## 3. 接口管理

### 3.1 接口操作

1. **添加接口**：
   - 输入接口名称、URL和请求方法
   - 添加参数和请求头
   - 编辑请求体JSON
   - 点击"保存接口"按钮保存

2. **编辑接口**：
   - 在接口列表中点击接口卡片
   - 修改接口信息
   - 点击"保存接口"按钮更新

3. **删除接口**：
   - 在接口列表中点击接口卡片
   - 点击"删除接口"按钮

### 3.2 批量操作

1. **多选接口**：
   - 在接口列表中勾选多个接口的复选框

2. **导出接口**：
   - 勾选要导出的接口
   - 点击"导出接口"按钮
   - 保存生成的JSON文件

3. **导入接口**：
   - 点击"导入接口"按钮
   - 选择要导入的JSON文件
   - 系统会自动添加导入的接口

## 4. 历史记录

### 4.1 查看历史

- 在"历史记录"页面查看所有过往请求
- 每条记录包含接口名称、请求方法、URL、请求头、请求体和响应信息

### 4.2 清空历史

- 点击"清空历史记录"按钮清除所有历史记录

## 5. 系统设置

### 5.1 基本设置

1. **协议**：选择HTTP或HTTPS
2. **基础URL**：输入API服务的基础URL，例如：`10.210.105.53:9000`
3. **请求超时**：设置请求超时时间（毫秒）
4. **最大历史记录数**：设置系统保存的最大历史记录数量

### 5.2 保存设置

- 点击"保存设置"按钮保存配置

## 6. 常见问题

### 6.1 请求失败

- 检查URL是否正确
- 检查网络连接
- 检查请求参数和请求头是否符合接口要求
- 检查请求体格式是否正确

### 6.2 接口导入失败

- 确保导入的文件是有效的JSON格式
- 确保JSON文件包含正确的接口结构

### 6.3 历史记录不保存

- 检查浏览器是否支持localStorage
- 检查最大历史记录数设置是否合理
